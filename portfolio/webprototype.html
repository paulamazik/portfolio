04.03.01 Web prototype

click to see the web prototype

Now it was time to bring everything we have done to life. After all the testing, we added or removed some things to make it all work better. And then we started working with HTML, CSS and Javascript again.

My website is a portfolio website for KEA students which allows them to show the work they have done and collab with companies. Even though it is not the best website someone could make and it didn't meet all the requirements, I have tried my best. It wasn't easy to code it all since the solutions we chose were hard for me to code, also we didn't have a lot of time which made me even more stressed. However, it definitely gave me the chance to improve my abilities to work under stress.

I had trouble redoing it after the given feedback which said that I didn't have the animation- wasn't sure where to put it and just did a simple moving message form. I also had a problem with creating the mobile version since it worked just fine on my devices, but turns out it doesn't on other's devices.

I have learned that even though you might think something is working, it doesn't mean it actually is- that's why it is really important to test your products. This project made me realise how structured coding this kind of projects is. It also made me realise that I really liked the UX part, not really the coding part. But I tried, failed, learned what my mistakes were and something about myself like the fact that I work better when I have more time to focus since this project made me feel really overwhelmed by the fact that I'm still a beginner when it comes to codig. But I'm not giving up yet, I'll try more times, fail and start again. :)
